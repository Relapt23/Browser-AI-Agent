# Browser AI Agent

AI-агент, который управляет браузером для выполнения задач. Вы пишете задачу в терминале — агент открывает браузер и выполняет её пошагово: ищет товары, заполняет формы, удаляет письма, оформляет заказы.

## Как это работает

1. Агент делает snapshot страницы — компактное представление интерактивных элементов
2. LLM анализирует snapshot и выбирает одно действие (click, type, scroll, navigate, wait)
3. Executor выполняет действие в браузере через Playwright
4. Цикл повторяется до завершения задачи

Перед чувствительными действиями (оплата, пароли, персональные данные) агент запрашивает подтверждение.

## Установка

```bash
# Клонировать репозиторий
git clone <repo-url>
cd browser-ai-agent

# Установить зависимости
uv sync
uv run playwright install chromium

# Настроить переменные окружения
cp .env.example .env
# Заполнить .env (см. раздел Конфигурация)
```

## Использование

```bash
# Передать задачу аргументом
uv run python run.py "найди погоду в москве на завтра"

```

### Примеры задач

```
зайди в яндекс почту и удали первые 10 сообщений
найди на wildberries кроссовки nike air max и добавь в корзину
зайди в деливери и закажи ангус чеддер-бекон, добавь в корзину
открой hh.ru и найди вакансии python разработчика в москве
```

## Конфигурация (.env)

### LLM

| Переменная | Описание |
|-----------|----------|
| `LLM_API_KEY` | API-ключ провайдера |
| `LLM_MODEL` | Модель (например `anthropic/claude-sonnet-4-6`, `openai/gpt-4o`) |
| `LLM_BASE_URL` | URL API (OpenAI-совместимый: OpenRouter, Azure, Ollama, proxy) |

### Браузер

| Переменная | По умолчанию | Описание |
|-----------|-------------|----------|
| `BROWSER_HEADLESS` | `False` | Запуск без GUI |
| `BROWSER_SLOW_MO` | `150` | Задержка между действиями (мс) |
| `BROWSER_VIEWPORT_WIDTH` | `1280` | Ширина окна |
| `BROWSER_VIEWPORT_HEIGHT` | `900` | Высота окна |
| `BROWSER_PAGE_TIMEOUT` | `8000` | Таймаут на действие (мс) |
| `BROWSER_MAX_STEPS` | `50` | Максимум шагов агента |
| `BROWSER_CDP_URL` | | URL для подключения к Chrome через CDP |

## Ключевые механики

### Snapshot

Вместо отправки HTML в LLM, агент извлекает из DOM компактный snapshot: список интерактивных элементов с их атрибутами (id, role, text, checked, visible). Каждый элемент получает уникальный `element_id` (e0, e1, ...), а snapshot — `snapshot_id`. LLM оперирует только этими id.

### Containers

Элементы группируются в логические контейнеры (список писем, таблица товаров). Контейнеры позволяют считать выделенные элементы и понимать структуру страницы.

### Selection Controls

Скрытые чекбоксы (hover-reveal) автоматически обнаруживаются и помечаются как `selection=item`. Executor делает JS-click для скрытых элементов, минуя CSS-ограничения.

### Валидация

- **Pre-validation** (`required_state`) — проверка перед действием (например, выделено ли нужное количество элементов)
- **Post-validation** (`expected`) — проверка после действия (target_checked, url_contains, text_visible)

### Защита от чувствительных действий

Паттерны URL и текста элементов для обнаружения оплаты, CAPTCHA, паролей. Агент запрашивает подтверждение пользователя перед выполнением.

## Tech Stack

- Python 3.11+
- [Playwright](https://playwright.dev/python/) — автоматизация браузера
- [OpenAI SDK](https://github.com/openai/openai-python) — совместимый с любым OpenAI-like API
- [Pydantic](https://docs.pydantic.dev/) — модели данных и конфигурация
- [Rich](https://rich.readthedocs.io/) — вывод в терминал
